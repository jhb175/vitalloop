# Apple Watch UI 设计说明

更新时间：2026-05-09 13:23

## 设计判断

Apple Watch 端不应是 iPhone 首页的缩小版。用户抬腕通常只会停留几秒，Watch 端应该回答三个问题：

- 我现在状态怎么样？
- 今天最该做哪件事？
- 我要不要快速补一条主观记录？

因此 Watch 端采用“状态 + 行动 + 快速记录 + 常驻组件”的结构。

- `figma-demos/body-coach-watch-ui-v1.html`：浏览器预览。

画板尺寸：

- Apple Watch 46mm：`416 × 496`

## 四个页面

### Watch A - 数据总览

用途：默认首页。

信息：

- 当前时间和电量。
- 小综合分状态，不作为主视觉。
- 心率变化。
- 活动能量。
- 睡眠。
- HRV。
- 步数。
- 下一步行动条。

设计原则：

- 手表首页优先看数据，不以评分圆环占据主视觉。
- 评分只做状态摘要，辅助理解，不抢数据表格位置。
- 使用多个小表格和小趋势图，避免单个大图表占满屏幕。
- 所有数据卡必须在 416 × 496 画板内完整显示，不能被圆角裁切。

### Watch B - 今日任务

用途：用户点开后直接执行。

信息：

- 今日 3 件事。
- 按恢复风险排序。
- 每条任务包含动作和理由。

第一版任务：

- 饭后快走 35 分钟。
- 23:30 前上床。
- 晚餐保持正常。

### Watch C - 快速记录

用途：补足算法无法从 HealthKit 获得的主观数据。

记录项：

- 压力
- 疲劳
- 饥饿

设计原则：

- 不用长表单。
- 不输入文字。
- 使用 0 到 10 的轻量评分控件。

### Watch D - 表盘 / Smart Stack

用途：常驻入口和提醒。

组件：

- 状态分组件。
- 睡眠提醒组件。
- Smart Stack 行动卡。

设计原则：

- 不承载完整解释。
- 只显示状态、下一步、提醒时间。

## 视觉语言

- 延续 iPhone v7 的深色 Liquid Glass 风格。
- 使用薄荷绿、蓝色、暖橙、淡紫作为功能状态色。
- 文字层必须高于背景光斑和玻璃反光。
- Watch 小屏上不使用大段解释文字。
- 卡片圆角更大，贴合 Watch 屏幕圆角。

## SwiftUI 实现建议

优先实现：

- `WatchStatusView`
- `WatchTaskListView`
- `WatchQuickLogView`
- `WatchComplicationPreview`

数据来源：

- 第一版用 iPhone 端生成的 `DailyBodySummary`。
- Watch 快速记录通过本地记录或 WatchConnectivity 同步回 iPhone。
- 第二阶段再考虑 Watch 端独立 HealthKit 汇总。

## 风险

- Watch 屏幕很小，不能塞入 iPhone 首页的数据表格。
- 过多玻璃反光会影响可读性。
- 表盘组件不能展示过多解释，否则会变成噪音。
- 主观记录如果超过 3 项，用户大概率不会长期使用。
