# Apple Watch 身体状态教练 App 项目进度

更新时间：2026-05-09 16:55

## 项目定位

开发一个 Apple Watch + iPhone App，用于身体状态监控、目标管理和个性化建议。当前英文名草案为 `VitalLoop`。

第一阶段重点不是拍照识别饮食，而是先做好“身体数据监测与总结算法”。减脂作为第一个目标场景，后续可扩展到增肌、减压、运动能力提升、技能训练等目标。

## 当前核心判断

- 产品内核：目标驱动的身体状态教练。
- 第一版目标：减脂。
- 第一版重点：身体数据总结、趋势判断、今日建议。
- 数据策略：本地优先，原始 HealthKit 数据默认不上传云端。
- 云端 AI：后续可作为增强能力，只处理用户明确授权后的摘要数据。
- 拍照饮食：不作为核心能力，可作为饮食记录的辅助入口。

## 身体数据监测方向

第一版监测指标：

- 活动能量
- 步数
- 锻炼记录
- 睡眠
- 静息心率
- HRV
- 体重
- 主观压力
- 疲劳感
- 饥饿感
- 饮食大致记录

算法目标：

- 今日身体状态评分
- 睡眠评分
- 恢复/压力趋势评分
- 活动负荷评分
- 体重趋势与目标进度
- 数据完整度评分

重要边界：

- 不做医疗诊断。
- 不声称准确检测压力。
- 热量消耗只作为趋势参考。
- 评分必须可解释，不能只给一个神秘分数。

## UI 方向

已确定先做 3 个首页 Demo 用于选择视觉方向。

### Demo A - Dashboard

专业仪表盘方向，信息密度最高，适合身体数据控制台。

### Demo B - Coach

教练建议优先方向，重点是让用户每天知道该怎么做。

### Demo C - Apple Native

Apple Health / Fitness 风格，原生感强，开发风险最低。

当前建议：

- MVP 可优先采用 Demo C 的原生风格。
- Demo A 的信息结构可作为后续高级仪表盘能力。
- Demo B 适合提高长期使用和行动转化。

### UI 反馈与 v2 调整

用户对 v1 UI 不满意，主要问题：

- Demo B 顶部整块黑色卡片太粗糙。
- Demo A 图表太单调，柱状图缺少设计感。
- Demo C 总结内容太弱，缺少有效信息。
- 整体没有充分体现参考图里的深色健康监控、彩色趋势图、时间轴数据卡片和 Watch 风格排版。

已根据参考图重新制作 v2：

- 深色玻璃质感界面。
- 更强的顶部状态总结。
- 身心指标卡片使用彩色折线趋势。
- 数据洞察页加入心率曲线、平均/最高/最低统计、时间轴条形图。
- Watch/专业卡片方向加入手表风格状态模块、圆环指标和双列专业图表。

### UI v3 / v4 调整

用户进一步反馈：

- 当前图表和数据只是占位，需要明确说明。
- 字体层级需要优化。
- 首页数据表格不应像二级页面的单个指标详情。
- 需要同时做深色模式和浅色模式。
- 颜色搭配需要更克制、更高级。

已制作 v3：

- 将首页改为“综合身体驾驶舱”。
- 首页包含综合评分、状态判断、核心信号、日内节律、身体雷达、体重趋势和今日建议。
- 将心率曲线、时间轴密集图表放到二级“指标详情”示例中。

已制作 v4：

- 同一套首页信息架构，分别输出深色模式和浅色模式。
- 深色模式：黑灰底 + 薄荷绿 / 蓝 / 暖橙 / 紫作为状态色。
- 浅色模式：冷白灰底 + 墨色文字 + 降饱和健康状态色。
- 当前 v4 是最新评审版本。

### UI v5 Liquid Glass

用户反馈 v4 的表格、颜色和字体方向可以，但卡片和背景质感不够好，希望使用 Liquid Glass 优化质感。

已制作 v5：

- 保留 v4 的首页结构、深浅模式、颜色和字体层级。
- 卡片改为 Liquid Glass 风格：半透明层、边缘高光、内部描边、柔和阴影。
- 背景加入可被玻璃折射的柔和彩色层。
- 底部导航改为浮动玻璃导航。
- chip、指标小卡、数据条增加玻璃质感和轻微发光。
- 真实 SwiftUI 落地时应使用 iOS 26+ `GlassEffectContainer`、`.glassEffect(...)`、glass button style，并为旧系统提供 material fallback。

### UI v6 Liquid Glass 修正

用户对 v5 注释：

- 底部导航像一个不协调的正方形，和手机底部圆角曲线不一致。
- 导航容器、图标和文字比例不对。
- 顶部玻璃质感廉价，像硬贴了一块高光。
- 卡片没有真正的悬浮感。
- 排版比例和空白利用不符合美学。

已制作 v6：

- 去掉廉价的大块高光，改为更克制的边缘高光和连续玻璃面。
- 顶部主卡改成更自然的 28px 圆角玻璃面板。
- 综合分圆环缩小，和右侧总结文字重新配比。
- 指标小卡、图表卡、建议卡统一更圆润的 22px 圆角。
- 底部导航改成贴合手机底部的半透明底板 + 胶囊导航，避免方块割裂。
- 压缩顶部和卡片间距，提升屏幕空间利用率。
- 当前 v6 是最新评审版本。

### UI v7 可读性修正

用户提醒：部分文字在背景图案和玻璃反光中看不清。

已检查并制作 v7：

- 发现 v6 的屏幕反光层、玻璃高光层和卡片 shine 层有可能干扰文字。
- 发现浅色/深色模式的部分二级文字偏淡，小字号说明在玻璃背景上可读性不足。
- 将装饰光斑、反光和卡片高光明确放到内容文字层后方。
- 降低整体反光和卡片高光透明度，避免文字被洗掉。
- 提升 `soft` / `muted` 文字颜色对比度。
- 顶部主文案区域加入轻量可读性底衬，保证文字不直接压在复杂纹理上。
- 新增 `figma-demos/ui-contrast-audit.md` 记录对比度问题和后续设计规则。

当前 v7 是最新评审版本。
