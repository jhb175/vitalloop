# UI Contrast Audit

更新时间：2026-05-09 12:49

## 检查结论

当前 v6 的主要问题不是字体本身，而是文字层和装饰层的关系不够严格：

- 屏幕整体反光层可能盖在内容上，导致白字发灰、浅色模式文字发虚。
- 玻璃卡片的高光层透明度偏高，文字密集区域容易被高光和背景光斑干扰。
- 二级信息文字使用了偏淡灰色，小字号状态说明在玻璃背景上可读性不足。
- 顶部主卡文案直接压在复杂玻璃纹理上，缺少稳定的阅读底面。

## 已应用到 v7 的解决方案

- 新增 `body-coach-ui-v7-contrast.html` 作为可读性修正版。
- 将 `screen::before`、`screen::after`、`glass-panel::before`、`glass-panel::after`、`card::before` 明确放到文字层后方。
- 降低屏幕反光、高光和卡片 shine 的透明度，保留质感但不干扰文字。
- 提升深色/浅色模式的 `--soft` 和 `--muted` 对比度，避免说明文字变成装饰性灰字。
- 在顶部主文案区域加入轻量 `hero-copy` 可读性底衬，帮助文字避开复杂玻璃纹理。
- 深色模式为标题和主说明增加非常轻的文字阴影，只用于分离背景，不做发光字效。

## 后续设计规则

- 主标题、状态判断、行动建议必须使用最高对比度文字色。
- 说明文字不能低于 11.5px，且不能使用低透明度灰色叠在玻璃背景上。
- 装饰光斑、反光、折射纹理只能在文字后方，不能覆盖文本层。
- 文字密集区域如果落在复杂背景上，必须加轻量底衬或降低背景复杂度。
- 浅色模式不要用过浅的灰蓝文字；深色模式不要只依赖透明度区分层级。
- 图表标签和时间轴文字按功能文字处理，不按装饰文字处理。

## SwiftUI 落地提示

在 iOS 26+ 使用 Liquid Glass 时，文字层应和 glass background 分离。建议结构是：背景氛围层、Glass 容器层、内容文字层。文字密集区域可以在 glass 容器内部增加半透明 readability scrim，但不要再叠一张明显的卡片。
