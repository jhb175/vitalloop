# Body Coach 下一阶段开发计划

更新时间：2026-05-09 12:49

## 当前基准

下一阶段以 `figma-demos/body-coach-ui-v7-contrast.html` 为 UI 参考基准。

产品第一版先做“身体状态监控 + 可解释评分 + 今日建议”，减脂只是第一个目标场景。拍照识别饮食不进入核心闭环，最多作为后续辅助记录入口。

## MVP 范围

### iPhone 端

- 今日身体总览：首页展示综合分、状态判断、关键指标、今日建议。
- 趋势页：睡眠、活动、恢复、体重趋势。
- 计划页：减脂目标、每周计划、今日任务。
- 记录页：体重、饥饿感、疲劳感、压力感、饮食大致记录。
- 设置页：HealthKit 权限、隐私设置、本地数据清理。

### Apple Watch 端

- 今日状态：综合分和一句状态判断。
- 今日任务：运动、睡眠、饮食提醒。
- 快速记录：疲劳、压力、饥饿、体重。
- 轻提醒：久坐、活动不足、恢复不足、睡前提醒。

### 暂不进入 MVP

- 自动拍照识别热量。
- 医疗级压力检测。
- 云端上传原始 HealthKit 数据。
- 复杂 AI 私教对话。
- 社交、排行榜、内容社区。

## 开发顺序

1. `BodyCoachCore`
   - 定义每日身体摘要输入模型。
   - 实现评分算法 v1。
   - 输出可解释原因和建议。
   - 用单元测试固定边界行为。

2. HealthKit Adapter
   - 读取活动能量、步数、锻炼、睡眠、静息心率、HRV、体重。
   - 转换为 `BodyMetrics`。
   - 不把原始 HealthKit 数据传到云端。

3. iPhone SwiftUI App
   - 按 v7 设计实现首页。
   - 先接模拟数据，再接 `BodyCoachCore`。
   - 深色/浅色模式同步落地。

4. watchOS App
   - 实现简化 Watch 首页和快速记录。
   - 与 iPhone 端共享核心评分结果。

5. 本地持久化
   - 保存目标、每日摘要、主观记录、体重记录。
   - 优先 SwiftData 或本地文件，避免第一版过早上云。

## 第一版核心模块

已创建 Swift Package：

- `BodyCoachCore/Package.swift`
- `BodyCoachCore/Sources/BodyCoachCore/BodyCoachCore.swift`
- `BodyCoachCore/Tests/BodyCoachCoreTests/BodyCoachCoreTests.swift`

核心输入：

- 睡眠分钟数和睡眠基线
- 活动能量和目标
- 步数
- 静息心率和基线
- HRV 和基线
- 体重和 7 天变化
- 主观压力、疲劳、饥饿
- 锻炼分钟数

核心输出：

- 综合分
- 睡眠分
- 恢复分
- 活动分
- 体重趋势分
- 数据完整度分
- 状态：strong / normal / caution / recovery
- 解释文案
- 今日建议

## 评分原则

- 分数必须可解释，每个低分项都要能说明原因。
- 评分是行为建议工具，不是医疗诊断。
- 热量和压力只用于趋势判断，不承诺绝对准确。
- 缺数据时降低数据完整度，不假装判断很准确。
- 恢复差时，建议优先恢复而不是继续增加运动量。

## Figma 交接规则

当前会话仍没有直接写入 Figma 的工具，因此下一步 Figma 操作采用两种路径：

- 短期：用 `figma-demos/body-coach-ui-v7-contrast.html` 作为视觉参考，必要时导出截图或 SVG 进 Figma。
- 正式：连接可写 Figma 工具后，创建页面 `Body Coach App V1`，输出 iPhone 首页、趋势页、计划页、记录页、Watch 首页。

Figma 设计落地时必须遵守：

- 深色和浅色模式同时维护。
- 所有文字必须通过对比度检查，不允许直接压在复杂玻璃纹理上。
- 玻璃反光、背景光斑、卡片高光必须在文字层后方。
- 首页是身体总览，不做成单个指标详情页。
- 图表数据在真实接入前必须标注为模拟数据。
